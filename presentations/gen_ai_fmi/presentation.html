Ask / Agent / Plan Modes — When to Use Each

Ask

What it does: Conversational query — AI reads context but takes no actions. Single turn.

When to use: Understanding, exploration, quick questions

Example: "What does this function do?" "Is there a performance issue in this query?"

Plan

What it does: Proposes changes, shows you the plan — doesn't execute until you approve

When to use: Complex/risky changes, schema changes, anything hard to reverse

Example: Large refactor, auth changes, multi-file deletions

Agent

What it does: Plans AND executes autonomously — reads files, writes code, runs tests, fixes failures

When to use: Feature implementation, refactoring, test generation

Example: "Implement the data quality check module"

Rule of thumb: Ask for understanding. Plan for caution. Agent for execution. Most people default to Ask for everything — the shift to Agent mode is where the large time savings come from.

24 / 60
Three modes. Three completely different workflows. Ask mode is conversational. The AI reads your context but does not take actions. Single turn. Use it when you want understanding. Plan mode proposes changes without executing them. You see exactly what the AI intends to do before it does anything. Use this for complex changes, schema changes, anything where a wrong execution would be painful to reverse. Agent mode plans and executes autonomously. It reads files, writes files, runs terminal commands, installs packages. It iterates — if it writes code and the tests fail, it reads the failure and fixes it. This is delegation, not assistance. The rule of thumb: ask for understanding, plan for caution, agent for execution. Most people default to Ask for everything. The shift to using Agent mode for implementation tasks is where the large time savings come from.

Prompting — The Multiplier

Effective prompt structure (5 components)

1. Role/context — "Working on the FM&I gas pricing model. Dataiku Python recipe."
2. Task — "Refactor calculate_forward_curve"
3. Constraints — "Without modifying the function signature or return type"
4. Output format — "Refactored code with inline comments explaining each change"
5. Edge cases — "Handle the case where the input DataFrame is empty"

"Here is my vague task: [X]. Generate a precise, well-structured prompt I can use to accomplish this."

The AI knows what information it needs to produce a good output. Ask it to specify that.

"What context would make this answer better?"
